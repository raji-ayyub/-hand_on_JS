Deployment to Vercel from GitHub

This lesson covers the usual GitHub to Vercel deployment flow for frontend projects, including preview deployments and production deploys.

Back to production track

What this gives you

  • Preview deployments for branch pushes and pull requests
  • Production deployments from your production branch
  • A live hosted URL for testing and sharing
  • A clean path to add a custom domain later

Before you start

  • A GitHub repository with the project pushed to it
  • A Vercel account
  • A project that already works locally
  • Correct GitHub permissions for the repository

Standard dashboard flow

  1. Push your project to GitHub.
  2. Sign in to Vercel.
  3. Click New Project.
  4. Under Import Git Repository, connect GitHub.
  5. Select the repository and click Import.
  6. Review build settings, root directory, and environment variables.
  7. Click Deploy.
After import, Vercel automatically creates preview deployments for branch pushes and production deployments for updates to the production branch.

Typical Git workflow before deployment

git switch -c feature/add-search-filter
# make your changes
git add .
git commit -m "Add search filter to the user directory"
git push -u origin feature/add-search-filter

Then open the pull request, test the Vercel preview URL, and merge when ready.

Common configuration points

  • Framework preset
  • Root directory
  • Build command
  • Output directory
  • Environment variables

For plain HTML, CSS, and JavaScript projects, the setup is often minimal.

Common issues

Issue Likely cause What to check
Repository missing in Vercel Wrong account or missing repo permissions GitHub account connection and repository access
Deployment succeeds but app is broken Wrong build, root, output, or env settings Project settings and build logs
Preview is correct but production is old Wrong production branch or wrong connected repo Project Git settings in Vercel

Good habits

  • Keep the production branch stable.
  • Use preview deployments before merging.
  • Store secrets in Vercel environment variables, not in source files.
  • Re-check deployment settings after moving folders inside the repository.

Practice prompts

  • Connect a small practice repository to Vercel.
  • Push a feature branch and confirm a preview deployment appears.
  • Merge it and confirm production updates.
  • Write down the exact branch that acts as production for your test project.
