Git fundamentals and core commands

Git gets much easier once you understand the basic flow: inspect, edit, review, stage, and commit.

Back to production track

Simple mental model

Git tracks snapshots of your repository over time.

  1. Check the state with git status.
  2. Review changes with git diff.
  3. Stage intended files with git add.
  4. Commit the staged snapshot with git commit.

Golden rule

If you are confused, run git status first. It tells you what branch you are on, what changed, and what Git expects next.

Core commands

git status
git diff
git diff --staged
git add path/to/file
git add .
git commit -m "Add responsive layout lesson"
git log --oneline --decorate --graph
git branch
git switch -c feature/layout-grid
git switch main
git restore path/to/file
git restore --staged path/to/file

Command guide

Command Use it when Effect
git status You want the current truth Shows branch, staged files, unstaged files, and untracked files
git diff You want to inspect unstaged edits Shows line-by-line changes not yet staged
git add file You are ready to stage a file Moves that file's changes into the staging area
git restore --staged file You staged the wrong file Unstages the file without deleting your edits
git restore file You want to discard local changes in one file Reverts that file back to the last committed version

Safe daily workflow

git status
git switch -c feature/form-feedback
# edit files
git diff
git add lessons/lesson_07_forms_and_validation/script.js
git diff --staged
git commit -m "Add validation feedback messages"

Practice prompts

  • Create a test repository and practice each command above once.
  • Stage the wrong file and recover with git restore --staged.
  • Make an accidental edit and discard it with git restore.
  • Write three stronger commit messages than update files.
