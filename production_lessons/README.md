# Production Lessons

This folder adds a second learning track for the work that turns beginner code into professional workflow.

These lessons focus on:

- production-ready frontend code practices
- Git fundamentals and everyday commands
- GitHub collaboration and pull request workflow
- common Git mistakes, error messages, and safe fixes
- deployment to Vercel from GitHub

## Recommended order

1. `production_lessons/production_lesson_01_production_ready_code_practices`
2. `production_lessons/production_lesson_02_git_fundamentals_and_core_commands`
3. `production_lessons/production_lesson_03_github_collaboration_and_pull_requests`
4. `production_lessons/production_lesson_04_common_git_errors_and_fixes`
5. `production_lessons/production_lesson_05_deployment_to_vercel_from_github`

## How to use this track

1. Open [production_lessons/index.html](index.html) in your browser.
2. Read the lesson `README.md` for the deeper explanation.
3. Copy the Git commands into a test repository, not your main work, when practicing risky scenarios.
4. Build the habit of checking `git status` before and after every important Git command.

This track is meant to sit beside the frontend lessons. Learn how to build the interface, then learn how to ship and maintain it responsibly.
