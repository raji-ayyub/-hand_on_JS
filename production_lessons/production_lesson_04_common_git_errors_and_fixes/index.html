Production 04

Common Git errors and fixes

Git problems feel much less scary when you know the usual causes and the safer recovery steps.

Back to production track

Start here when something feels wrong

git status
git branch
git log --oneline --decorate --graph -10
These commands answer the three most important questions: Where am I, what changed, and what happened recently?

Common problems and safe fixes

Problem Likely cause Safer fix
Wrong file is staged You used git add . too broadly git restore --staged path/to/file
Need to discard local edits in one file You changed a file by mistake git restore path/to/file
Push rejected The remote branch changed first git fetch origin, then pull or rebase, then push again
Merge conflict Two branches changed the same lines differently Resolve markers manually, stage the file, then continue
Wrong commit message The latest commit message is weak or incorrect git commit --amend -m "Better message"

Examples

# Unstage one file
git restore --staged src/app.js

# Discard local changes in one file
git restore src/app.js

# Update local history before pushing
git fetch origin
git pull --rebase origin main
git push origin feature/search-filter

# Fix the latest commit message
git commit --amend -m "Improve user directory filter state"

What to avoid unless you truly mean it

  • git reset --hard can permanently discard uncommitted work.
  • Force-pushing shared branches can disrupt teammates.
  • Randomly retrying commands without checking git status usually creates more confusion.

Practice prompts

  • Practice staging the wrong file and undoing it safely.
  • Create two branches with conflicting edits to the same line and resolve the conflict.
  • Make a weak commit message, then improve it with git commit --amend.
  • Write your own "Git panic checklist" with the three commands you run first.
